SpringCloud Alibaba

文档地址： https://github.com/alibaba/spring-cloud-alibaba

中文文档地址：https://github.com/alibaba/spring-cloud-alibaba/blob/2.2.x/README-zh.md

1.添加一下依赖来进行版本控制，最新依赖进中文文档查询，后续组件无需版本号

nacos

nacos 服务注册与发现

1.添加 Nacos Discovery 依赖进行服务注册与发现，Nacos Discovery 适配了 Netflix Ribbon，可以使用 RestTemplate 或 OpenFeign 进行服务的调用。

2.添加 spring.cloud.nacos.discovery.server-addr地址 和 服务名称 配置到 application.yml 中

3.添加注解 @EnableDiscoveryClient ，让 nacos注册中心发现

4.启动 nacos注册中心 startup.cmd

5.启动服务 gulimall-member,发现gulimall-coupon服务已经注册到nacos

feign远程调用

1.添加 openfeign依赖

2.编写服务方服务

3.在消费方添加接口

5.在消费方主启动类添加注解@EnableFeignClients，支持远程服务调用

6.启动两个服务，到浏览器输出地址 localhost:8000/gulimallmember/member/coupon/list，成功

nacos 配置中心

1.导入依赖

2.在应用的 /src/main/resources/bootstrap.properties 配置文件中配置 Nacos Config 元数据

bootstrap.properties 优先级高于 application.properties

3.测试获取 application.properties 配置中的数据

controller

浏览器输入地址 http://localhost:7000/gulimallcoupon/coupon/test 测试成功

4.使用配置中心配置，在 nacos操作页面 ->配置列表->新建配置，Data Id 为 gulimall-coupon.properties（应用名.properties）

5.在 controller 中添加注解 @RefreshScope 注解可实时更新配置（热更新）

6.在代码不变的情况下配置中心的配置 优先级高于 application.properties 中的配置

即远程大于本地，测试大于共享

命名空间 namespace

命名空间（namespace）主要是进行配置隔离

1.例 将配置中心按服务进行配置隔离，创建命名空间 coupon，ware，order，product，member

1）在 nacos服务页面 命名空间->新建命名空间

2）如需使用特定配置，需要到 bootstrap.properties 中配置命名空间Iid即可,下实例配置的是 coupon服务id

配置集

所有配置的集合

1.我们可以拆分配置集将 application.yml 中的配置集分门别类进行管理如拆成：mybatis，datasource，other等并分组管理

2.在到 bootstrap.properties 中配置 扩展配置 extension-config

配置集ID

Data Id : 类似于文件名

配置分组 Group

1.我们可以用分组的方式将一个服务空间下分为不同环境的组配置，如：dev，prod，test，默认情况下 使用 DEFAULT_GROUP 组

1）在 nacos服务页面 创建配置是 可以编辑组，或在已有配置选定进行克隆，修改组名和相应配置即可

2）想要使用组，需要到 bootstrap.properties 中配置组名,就可以在当前服务配置中切换为开发环境配置
